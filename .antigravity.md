# .antigravity.md — FinTrack Project Context

## 1. Project Overview & Architecture

**Finman** — Personal finance web app with OCR-based transaction recognition from bank notification screenshots. Target users: individuals tracking expenses + small merchants doing bulk receipt scanning.

**Monorepo structure:**
```
/be   — Node.js REST API + business logic + DB access
/fe   — React SPA + PWA (start_url: /scan)
/ocr  — Python FastAPI microservice for image/PDF extraction
```

**Request flow:**
```
FE → BE (/api/v1/*) → Supabase PostgreSQL
FE → BE (/api/v1/ocr/*) → OCR Python service → Gemini/Qwen → BE confirms → DB
```

## 3. Core Conventions (CRUCIAL — Never Violate)

### Amount / Money
- **Always `BigInt` (VND), never `float`**
- Always positive (unsigned)
- All arithmetic must use BigInt operators
- Example: `BigInt(85000)` not `85000.0`

### Soft Delete
- All deletable entities have `deletedAt: DateTime?` field
- **Every query MUST filter** `where: { deletedAt: null }`
- Never hard delete user data (except 6-month cron cleanup for archived wallets)
- Archived wallets use `archivedAt` — separate from `deletedAt`

### Atomic Balance Recalculation
- **Any operation that changes `wallet.currentBalance` MUST be wrapped in `prisma.$transaction()`**
- Pattern for POST transaction: `create tx → update wallet balance → commit`
- Pattern for PATCH transaction: `reverse old amount → apply new amount → commit`
- Pattern for DELETE transaction: `soft delete → reverse balance → commit`
- Pattern for goal deposit: `deduct wallet → credit goal → commit`
- Pattern for goal delete: `create refund income tx → credit wallet → mark abandoned → commit`

### Timestamps
- All timestamps: **ISO 8601 UTC**
- Use `new Date()` for `now()` values
- `transaction_date` is user-supplied date (not auto)

### API Response Format
**Success:**
```json
{ "success": true, "data": <T>, "message": "string" }
```
**Error:**
```json
{ "success": false, "message": "string", "code": "string" }
```

### HTTP Status Codes
| Code | When |
|---|---|
| 200 | OK |
| 201 | Created |
| 400 | Validation error (Zod) |
| 401 | JWT missing / expired |
| 403 | Forbidden (e.g., editing another user's data or default category) |
| 404 | Resource not found |
| 409 | Conflict / duplicate (e.g., same budget period, existing email) |
| 422 | Business logic error (e.g., insufficient balance) |
| 500 | Unhandled server error |

### Auth
- All endpoints require `Authorization: Bearer <accessToken>` except `POST /auth/register` and `POST /auth/login`
- `req.user` is attached by `authMiddleware` and contains `{ userId, email }`
- Refresh token stored in `users.refreshToken` column in DB; cleared on logout

### Category Ownership
- `userId = null` → system default category (read-only for all users)
- `userId = <uuid>` → user's custom category (owner-only edit/delete)
- Cannot edit or delete default categories (403)

### Wallet Archive Logic
- `currentBalance > 0` → `DELETE /wallets/:id` sets `archivedAt = now()` (not `deletedAt`)
- `currentBalance = 0` → `DELETE /wallets/:id` sets `deletedAt = now()`
- API docs note: client sends `?force=true` to confirm archive with balance

### `is_default` Wallet
- Only one wallet can have `isDefault = true` per user at any time
- Setting a new default must atomically unset the old one: `prisma.$transaction([updateMany isDefault=false, update target isDefault=true])`

### Folder / Layer Rules (BE)
```
routes    → define endpoints + Zod validation middleware + requireAuth
controller → extract from req, call service, return sendSuccess/next(error)
service   → all business logic, throws AppError, no req/res
repository → all Prisma queries, no business logic
```
- Controllers must NOT contain business logic
- Services must NOT import `req` or `res`
- Never import one module's service directly into another module's service (pass as param instead)

### FE Conventions
- Components never call `axios` directly — always through `src/features/*/api/*.api.ts`
- API hooks live in `src/features/*/api/` using React Query `useQuery` / `useMutation`
- Auth state read via `useAuthStore` (Zustand); outside components use `useAuthStore.getState()`
- After any mutation that affects wallet balance, invalidate `QUERY_KEYS.WALLETS` and `QUERY_KEYS.DASHBOARD`
- All `input`, `select`, `textarea` must have `font-size: 16px` minimum (prevents iOS Safari zoom)
- Bottom tab bar must use `padding-bottom: env(safe-area-inset-bottom)` (iOS home indicator)

### OCR Conventions
- Never store original image files — only `extractedText` (raw OCR string) saved to DB
- MD5 cache checked before any LLM call
- PDF files use `pdfplumber` and bypass LLM entirely
- `scanContext` ("expense" | "income") from user button is the initial type signal; NLP keywords override only if confident
- Confidence score: 0.0–1.0 based on ratio of successfully extracted fields
- Unrecognized bank → return `bankDetected: null`, FE shows manual dropdown

## 6. Key Business Rules Quick Reference

```
# Register side effect
POST /auth/register → creates user + creates default wallet "Ví chính" (isDefault=true) atomically

# Delete wallet with balance
DELETE /wallets/:id
  currentBalance > 0 → set archivedAt (409 warning, needs ?force=true per API docs)
  currentBalance = 0 → set deletedAt

# Budget alert dedup
After each transaction write → calculate spent in current period
  ≥80%: check alert80SentAt in current period window → if not sent → set flag + return budget_alert
  ≥100%: always return exceeded alert (no dedup)
  Period reset: clear alert80SentAt at start of new period

# Recurring detection trigger
After POST /transactions:
  Query: same userId + merchant + amount, transactionDate within ±5 days of 30-day interval, ≥2 occurrences
  If pattern found + no existing active rule + not snoozed → insert recurring_rules (isActive=false)

# Goal delete refund
DELETE /goals/:id → prisma.$transaction([
  create income transaction (source='system', note='Hoàn tiền từ goal [name]'),
  increment wallet.currentBalance by goal.currentAmount,
  update goal status='abandoned', deletedAt=now()
])

# OCR confirm saves
POST /ocr/confirm → prisma.$transaction([
  create transaction (source='ocr', extractedText=rawText),
  update wallet.currentBalance
]) → check budget alerts → return { transaction, wallet, budget_alert }
```
